01. Project DNA

The Purpose

An AI-powered knowledge-base agent for researching Azure DevOps work items and source code. It solves the disconnection between ticketing systems and local repositories by creating a unified context window for LLMs.

Tech Stack

Node.js v22+ TypeScript 5.7.3 MCP SDK 1.0.4 Azure DevOps API Git / Ripgrep

Core Capabilities

  • Unified Context: Correlate tickets, wikis, and 30+ repositories.
  • Local Knowledge: Shallow clones for instant local ripgrep search.
  • Dual Agent Support: Native compatibility with Gemini CLI and Claude Code.
  • Background Sync: Non-blocking async updates for large scale clones.

03. Key Logic Flows

Research Flow: /topic-work-item

1

Context Retrieval

Call Azure DevOps MCP to fetch work item metadata (description, acceptance criteria, comments).

2

Keyword Extraction

AI extracts technical keywords (e.g., class names, error codes) to form codebase search queries.

3

Multi-Repo Ripgrep

Search tool executes parallel ripgrep across 30+ cloned repos in .source/.

4

Technical Deep-Dive

AI reads specific code snippets, correlates with wiki docs, and generates the final analysis.

05. Developer Quickstart

1. Prerequisites & Environment

# Install Node.js v22+ and build-essential for ripgrep support
$env:AZURE_DEVOPS_EXT_PAT = "your_pat_here"

# Build the MCP server
cd mcp-server
npm install
npm run build

2. Repository Strategy

Modify .gemini/settings.json to configure the sync scope:

{
  "repositories": {
    "azureDevOpsOrg": "DprGroupLtd",
    "includedRepos": ["Tools.Topic", "DPR.Platform", "Core.Library"],
    "includedWikis": ["Product.Guide"]
  }
}
